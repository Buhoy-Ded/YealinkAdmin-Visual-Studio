# YealinkAdmin

Локальная админ-панель для управления IP-телефонами Yealink в корпоративной сети.

Проект запускается как ASP.NET Core / Blazor Server приложение и работает с телефонами напрямую по HTTP/HTTPS из локальной сети оператора.

## Стек

- .NET 8
- ASP.NET Core
- Blazor Server / Interactive Server
- Windows x64
- JSON-хранилища рядом с исполняемым приложением

Проект намеренно остается Interactive Server, не WASM. В опубликованном single-file EXE WASM уже давал проблемы со статикой и обработчиками кнопок.

## Возможности

- Сканирование телефонов по подсетям выбранной локации.
- Потоковое добавление найденных телефонов в UI во время сканирования.
- Кэширование списка телефонов и статуса в JSON.
- Отдельная авторизация приложения: пользователи, сессии, последний вход.
- Отдельный админ-режим для учетных данных телефонов.
- Локации с привязкой телефонов по маскам подсетей.
- Аудит действий пользователя и экспорт логов в TXT.
- Статус телефона, конфигурация аккаунтов, import/export CFG.
- Перезагрузка выбранных телефонов.
- История звонков для поддерживаемых моделей.
- Панель расширения для моделей, у которых она обнаружена.
- Автоисправление Action URI через загрузку минимального CFG.

## Поддерживаемые семейства

### T30P / T3x

Старый web/servlet flow:

- `phonecfg`
- legacy RSA login
- servlet endpoints
- legacy CFG import/export/reboot

### T43U / T46U / T57W / W70B

Modern API flow:

- HTTPS
- RSA login через `/api/common/info?p=Login`
- `/api/auth/login`
- CSRF token
- `/api/common/info?p=StatusGeneral`
- modern CFG import/export/reboot

```powershell
dotnet run
```

## Данные

Файлы создаются рядом с EXE через `AppContext.BaseDirectory`.

Основные файлы:

- `phones.json` - найденные телефоны, кэш статуса и конфигурации.
- `app-users.json` - пользователи приложения, пароли только в виде hash.
- `app-locations.json` - локации и правила подсетей.
- `app-audit-log.json` - журнал действий пользователя.
- `exports/` - выгруженные CFG и TXT-экспорт логов.
- `generated/` - временные delta CFG для отправки изменений в телефон.
- `debug/` - raw-диагностика API, если сохранялась из UI.

Нельзя сохранять на диск:

- admin-пароль телефонов;
- cookies телефонов;
- CSRF/session tokens телефонов;
- пароли из CFG/HAR.

Admin-креды телефонов живут только в памяти текущей сессии через `SecureCredentialStorage`.

## Авторизация

В проекте две разные сущности:

- Пользователь приложения - вход в YealinkAdmin.
- Админ-режим - временные admin-креды телефонов.

Пользователи управляются на странице `Настройки -> Пользователи`.

После входа пользователь выбирает локацию в левом нижнем блоке. Выбранная локация влияет на список телефонов и область сканирования.

## Локации и подсети

Локации находятся в `Настройки -> Локации`.

Поддерживаемые правила:

```text
10.6.10.*
10.6.10.0/24
10.6.10.136
```

Для сканирования:

- `10.6.10.*` преобразуется в `10.6.10.0/24`;
- точный IP преобразуется в `/32`;
- CIDR используется как есть.

Дефолтная основная локация использует:

```text
10.6.10.*
```

## Аудит

Журнал доступен в `Настройки -> Логи`.

Логируются:

- вход, выход и истечение сессии;
- неуспешный вход;
- смена локации;
- создание, изменение и удаление пользователей;
- создание, изменение и удаление локаций;
- сканирование и обновление списка;
- ручная проверка IP;
- удаление телефонов из списка;
- перезагрузка телефонов;
- загрузка, сохранение и очистка аккаунтов;
- import/export CFG;
- работа с панелью расширения;
- загрузка истории звонков;
- открытие веб-интерфейса телефона.

TXT-экспорт сохраняется в:

```text
exports/audit/
```

## Сканирование

Сканирование выполняет `YealinkScanner`.

Сейчас найденные телефоны добавляются в UI сразу во время сканирования, а не после завершения всего цикла.

Во время сканирования кнопки:

- `Сканировать новые`
- `Обновить список`

остаются неактивными.

## Конфигурация телефонов

Вкладка `Config`:

- загружает параметры аккаунтов из телефона;
- поддерживает несколько аккаунтов на modern-моделях;
- T30P остается с legacy-путем;
- изменения отмечаются звездочкой;
- при сохранении формируется delta CFG только с измененными параметрами;
- delta CFG отправляется в телефон;
- локальный кэш телефона обновляется параллельно с изменением на телефоне.

Отдельно есть import/export полного CFG.

## Панель расширения

Для телефонов с обнаруженной панелью появляется вкладка `Панель расширения`.

Логика:

- 3 страницы;
- по 20 клавиш на странице;
- слева нечетные, справа четные;
- тип клавиши для быстрого вызова: `13`;
- вызов идет от имени первого аккаунта.

## История звонков

Вкладка `История звонков` доступна для поддерживаемых моделей, кроме W70B.

Есть:

- загрузка истории;
- поиск;
- сортировка;
- нормализованный табличный вид.

## Action URI

Если телефон отвечает `403 Forbidden`, он добавляется в список и помечается как требующий настройки Action URI.

Для исправления используется минимальный CFG:

```text
features.action_uri_limit_ip = any
features.show_action_uri_option = 0
```

После успешного получения данных флаг ошибки должен сниматься.

- `Services/Infrastructure/AppUserStore.cs` - пользователи приложения.

- `Services/Infrastructure/AuditLogStore.cs` - аудит.

- `Services/Web/YealinkConfigManager.cs` - legacy config operations.
